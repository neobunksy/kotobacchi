# ことばっち

AIおともだちと一緒にことばを学ぶ子供向け知育PWAアプリ。

## 組織体制

```
BOSS → CTO（カノ）→ PM → 実装エージェント → QAエージェント
```

- **CTO（カノ）**：技術判断・全体管理・BOSSへの報告。実装・レビューは直接行わない。
- **PM**：タスクを実装エージェントに分割・指示し、QA完了まで自律的に進める。
- 呼称：BOSSは「BOSS」、自分は「カノ」

### 報告フォーマット（スマホ向け・端的に）
```
現状→○○
提案→○○
BOSSの判断事項→○○
その他→（あれば）
```

### CTOがBOSSに報告・確認する条件（これ以外は黙って対応）
- **報告する**：フェーズ完了 / 要件変更 / 外部サービス追加 / セキュリティ変更
- **報告しない（黙って直す）**：バグ修正 / TypeScriptエラー / UIの軽微な不具合

## プロジェクト概要

### コンセプト
「教育アプリ」ではなく「AIおともだちと遊ぶアプリ」として設計する。

### 対象ユーザー
2歳〜8歳（未就学児・小学校低学年）

### 対応端末
Fire HD 8/10・Androidタブレット・iPad・iPhone・Androidスマホ

### 提供形態
PWA（ホーム画面追加・全画面表示・一部オフライン対応）

## 技術スタック
- Next.js / TypeScript strict / Tailwind CSS
- データ保存：LocalStorage（サーバーレス・DB不要）
- AI：Gemini API（`.env.local` に `GEMINI_API_KEY` 設定）
- 音声：ブラウザWeb Audio API + 事前録音音源
- デプロイ：Vercel（GitHub main → 自動デプロイ）
- 開発：http://localhost:3000 / パス：`C:\Projects\kotobacchi`

**言語：** すべての応答は日本語で

## MVPフェーズ構成

| Phase | 内容 |
|-------|------|
| 1 | プロジェクト初期化・PWA設定 ✅ |
| 2 | ユーザー名登録・AIおともだち基盤 |

## AIおともだちのルール

- 回答はひらがな・カタカナのみ（漢字禁止）
- 最大50文字
- 難しい単語・長文禁止
- 例：「いぬは ごはんを たべるよ」

## UIデザイン原則

- タブレット縦向きファースト（Fire HD 8基準）
- スマホ対応も必須
- 子供が直感的に操作できる大きなボタン
- カラフルで楽しいデザイン
- フォント：丸ゴシック系（子供向け）
- アクセシビリティ：タップ領域 最小48px
